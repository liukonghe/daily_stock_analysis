# 策略版本冻结协议
# Strategy Version Lock Protocol
# 版本: V1.0 | 生效日期: 2026-06-30

## 核心原则

本仓库的策略文件是本系统的"宪法"。策略规则优先于 LLM 的自由判断。

## 版本冻结规则

### 当前版本：V1.0

- 冻结期：2026-06-30 至 2026-07-30（至少 20 个交易日）
- 冻结期内，以下文件不得修改核心逻辑：

| 文件 | 策略 | 版本 |
|------|------|------|
| `strategies/00_chokepoint_main_board.yaml` | A股主板过滤 | V1.0 |
| `strategies/01_chokepoint_core.yaml` | 瓶颈投资核心 | V1.0 |
| `strategies/02_chokepoint_robot_chain.yaml` | 机器人链 | V1.0 |
| `strategies/03_chokepoint_ai_computing.yaml` | AI算力链 | V1.0 |

### V1.0 允许的修改

- 修正错别字、格式错误
- 更新股票代码示例
- 更新评分数值（标注原因）
- 新增"暂不碰"条件（不能删除已有条件）

### V1.0 禁止的修改

- 新增/删除选股维度
- 修改三道门逻辑
- 修改一票否决条件
- 修改操作判断标准词定义
- 修改产业链拆解层级
- 未经复盘数据支撑的评分调整

## 版本升级规则

### V1.0 → V1.1（小版本）

条件：
- 至少 10 个交易日数据
- 至少 20 条决策记录
- 发现明确的 bug 或遗漏
- 用户确认

### V1.x → V2.0（大版本）

条件：
- 至少 20 个交易日数据
- 至少 50 条决策记录（含复盘统计）
- 有明确的胜率/收益/回撤数据支撑
- 新规则经过历史回测验证
- 用户确认并签字

## 决策留底规则

每次分析产生的决策信号，由 daily_stock_analysis 的 `DecisionSignal` + `DecisionSignalOutcome` 系统自动记录。

复盘周期：
- 每日：检查昨日推荐是否触发入手条件/失败信号
- 每周：统计胜率、方向正确率、是否跑赢指数
- 每月：评估策略整体有效性，决定是否升级版本

## 回测规则

新规则在加入策略前，必须：
1. 在历史数据上回测至少 3 个月
2. 与 V1.0 策略对比：胜率、收益、最大回撤
3. 提供回测报告（包含样本量、时间段、市场状态）

## 修改流程

```
1. 提出修改建议 → 写入候选规则池（candidate_rules.md）
2. 回测验证 → 附回测报告
3. 用户审核 → 确认是否采纳
4. 修改 YAML → 更新版本号 → 更新本文件
5. Git Commit → PR Review（如适用）
```

## 禁止行为

- ❌ 在未冻结期结束前修改核心策略逻辑
- ❌ 因为单次/短期市场变化临时调整策略
- ❌ 未经回测验证的新规则直接加入策略
- ❌ LLM 自由发挥覆盖策略规则
- ❌ 为凑数量推荐不符合条件的标的
- ❌ 推荐非主板标的（不管逻辑多强）
